## Contributing to Documentation

When adding new features or making significant changes, please update the relevant documentation:

- **New features**: Update README.md and relevant guides
- **Environment changes**: Update ENVIRONMENT_SETUP.md
- **Deployment changes**: Update DEPLOYMENT_WORKFLOW.md and DEPLOYMENT.md
- **Architecture changes**: Update ARCHITECTURE.md
- **New commands**: Update QUICK_REFERENCE.md

## Documentation Standards

- Use clear, concise language
- Provide code examples for all procedures
- Include troubleshooting sections
- Cross-reference related documents
- Keep documentation up-to-date with code changes
- Use consistent formatting and structure

## Need Help?

If you can't find the answer in the documentation:

1. Check the troubleshooting sections in relevant guides
2. Search existing GitHub issues
3. Create a new issue with your question
4. Contact the development team
